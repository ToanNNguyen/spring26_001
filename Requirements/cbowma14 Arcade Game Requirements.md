I chose Donkey Kong as my game.

•	Mandatory - Functional Requirements
o	The game MUST display a controllable player character capable of horizontal movement across platforms.
o	The game MUST allow the player character to perform a jump action.
o	The game MUST spawn rolling obstacles that move from the top of the level toward the bottom.
o	The game MUST detect a collision event involving the player character.
o	The game MUST decrement the player’s life count by one when a collision event occurs.
o	The game MUST display a visible score during gameplay.
o	The game MUST increase the score when an obstacle is successfully avoided.
o	The game MUST include a level goal for each round.
o	The game MUST end the current round when the level goal is reached.
o	The game MUST limit the player to a fixed number of lives per session.
o	The game MUST restart the current level after a life is lost.

•	Mandatory - Non-Functional Requirements
o	The game MUST respond to player input within 0.2 seconds.
o	The game MUST maintain a minimum frame rate of 30 frames per second during gameplay.
o	The game MUST operate continuously for at least 30 minutes without crashing.
o	The game MUST display gameplay elements at a minimum resolution of 1280 × 720.

•	Optional - Functional Requirements
o	The game MAY include multiple levels with increasing difficulty.
o	The game MAY include collectible power-ups that temporarily enhance player abilities.
o	The game MAY include bonus items that increase the player’s score when collected.
o	The game MAY include enemy characters that actively pursue the player.
o	The game MAY include a leaderboard displaying high scores.
